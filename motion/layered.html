Layered animation

Layers in this context are not related to visual stacks of elements. Layered animation refers to the animating of various different properties across multiple objects within a shared hierarchy.

Layers are what give an animation depth. This is one of the key things which make an animation nice to look at.

This animated thumbnail looks pretty nice! To the user it reads as 1 animation: the thumbnail appears and voila. However, this animation consists of several individual animating elements. Each animated with their own speeds and animation curves.

Note: even though the 'tail' of the animation is quite long, the element can be considered ready for interaction quite quickly. You can almost say that in cases like this you can divide the animation into a primary and secondary category.

  • Primary being the scaling animation with which the frame appears.
  • Secondary being the long tail animations which subtly ease out.

When working with UI, you'll most likely be dealing with object hierarchies. These hierarchies are ideal for animations such as this one. Parent transformations directly translate to the child objects as well. Working your way from top (parent) to bottom (children) is a great way to iterate through animation designs.

tl;dr animate all the things. Take into consideration primary and secondary layers of animation.
